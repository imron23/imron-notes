📂 Command & Feature Summary

Feature Share & Public View

Pengguna meminta fitur URL Share Publik. Sistem membangun modul ShareModal dan PublicViewPage, dengan daftar tautan aktif di Settings.

UI Revamp Dark Mode Global

Memperhalus mode gelap agar memengaruhi seluruh aplikasi, tidak hanya kanvas. Memodifikasi CSS global (Background #1f2937) agar popup, input, dan sidebar selaras.

Feature Auto-Backup & Persist

Kebutuhan auto-save sehingga ketikan tidak hilang saat restart docker/browser. Modul Zustand Persist Middleware berhasil diintegrasi.

UI Revamp VS Code Styled Blocks

Mengubah tampilan blok kode bawaaan agar tampil seidentik gaya Github/VS code. Ditambah pembaruan visual panel Stabillo/Color Picker.

Feature Search Sorting

Mencari kata di seluruh isi note dengan cermat, bukan sekadar di judul. Juga diurutkan menurut tanggal (descending).

Feature PDF Auto-Download

Penggantian dari `window.print` ke `html2pdf.js`. Hanya meng-ekspor *current view/wrapper* secara langsung tanpa intervensi dialog browser.

🐛 Bug Encountered & Resolved

  • 🐞

    Pop-up Slash Menu (/) Terpotong (Scroll Bug)

    Penyebab: Elemen `

    ` memiliki properti `overflow: hidden`, sehingga saat Floating-UI milik BlockNote (bertipe portal) ditarik ke bawah layar, posisinya tergunting batasan layouting CSS.
    Solusi: Membuang `overflow: hidden` dari wrapper dokument dan mengandalkan Window Scrolling alami HTML5.

  • 🐞

    Dark Mode Teks Hitam di Latar Hitam

    Penyebab: Node Editor dari dependensi `@blocknote/mantine` di *hard-code* pada mode `light`, mengabaikan tema gelap UI di sekitarnya.
    Solusi: Menyambungkan prop `theme` editor ke dalam Global Store `isDarkMode` secara langsung.

  • 🐞

    Public Share Page Hanya Muncul Judul Tanpa Isi

    Penyebab: Fungsi perender memanggil key array `doc.blocks` yang bernilai *null*, karena data tersebut sebetulnya disimpan di key `doc.content`.
    Solusi: Modifikasi pemanggil `renderBlocks((doc).content)`.

  • 🐞

    Search Modal tidak dapat menemukan kata dalam kontent

    Penyebab: Sama seperti fungsi perender, loop teks di SearchModal memedah `doc.blocks` bukan `doc.content`.
    Solusi: Stringify konten aktual dan menambahkan filter sort berdasar kapabilitas parameter createdAt.
